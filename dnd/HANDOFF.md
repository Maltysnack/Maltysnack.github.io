# dnd HANDOFF

Living scratchpad for the `/dnd/` character sheet subsystem. Read first thing when working on dnd; update last thing.

The root `HANDOFF.md` covers the rest of the site and points here for anything dnd.

## Current state (as of 18-05-2026)

- 5 live characters: Grosh (Triton Paladin Ancients 5), Shieka (Giant Warlock Fiend 5), Jhotan (Forest Gnome Battle Master 5), Japlan (Standard Human Sorcerer Draconic Brass 5), Sage Greenwood (Half-Elf Druid Shepherd 5, added today).
- Engine reads JSON per character, renders to a single-screen sheet with wiki-linked names everywhere.
- Upload flow: visitor drops a filled WotC PDF on `/dnd/`, Vercel function `api/upload.js` opens a PR. Wren polishes per `POLISH-PROCESS.md`, maltysnack merges, build action regenerates `index.json` + per-character HTML shim.
- Update flow: live sheet has an Update Sheet block; freeform "what changed" + state, `api/update.js` opens an update PR. Same polish + merge loop.
- Two PDF buttons on each sheet: **Sheet PDF** (WotC fillable export via pdf-lib) + **Print** (browser print -> save-as-PDF of the live rendered sheet).

## Wiring + services

- **Vercel project**: `dnd-upload` (team `maltysnacks-projects`). Production URL `dnd-upload-maltysnacks-projects.vercel.app`. Auto-deploy from GitHub not connected; redeploy manually via `vercel deploy --prod --yes` from repo root after touching `api/`.
- **GitHub PAT** env var on Vercel: stored as `dnd_upload` (not `GITHUB_TOKEN`). Function reads any of `GITHUB_TOKEN | dnd_upload | DND_UPLOAD | GH_TOKEN`.
- **Build action**: triggers on push to main with paths `dnd/characters/*.json` (excl. index.json + README.md). Retries push 3x on race rejection. Runs `node dnd/scripts/build-dnd-index.js`.

## Decisions worth not re-litigating

- **Per-character HTML filename = slug-word1-word2.html** (not query-string routing).
- **No backend for the static site**. Cross-device state sync deferred.
- **PR-based human review** chosen over auto-merge. The merge click is the spam protection.
- **Wren = the AI's name on this repo.** Co-authored-by `Wren <wren@maltysnack.dev>`.
- **Pip vs counter rule**: max <= 8 -> pip, max > 8 -> counter. Coded in `class-resources.js` via `dieType()`.
- **Wiki links**: Google search restricted to `dnd5e.wikidot.com` for non-spell items. Direct deep-link for spells. Homebrew detected by word "homebrew"/"hb" -> no link.
- **Feature grouping**: optional `group` field on features. Consecutive same-group features cluster under a wiki-linked header.
- **HP rule = fixed-die, not avg-rounded**: lvl1 = max + CON, then per level = `fixed + CON` where fixed = d6/4, d8/5, d10/6, d12/7. Example: lvl 5 d8 druid CON -1 -> 7 + 4*(5-1) = 23.

## Gotchas

- **pdf-lib `setText()` does NOT regenerate appearance streams** on existing form fields. Always call `form.updateFieldAppearances()` before save, or AC/Init/Speed render the cached appearance instead of the new value.
- **Parser Race-dropping bug** (the `Race ` field with trailing space was in the `handled` set but never read into the JSON). Fixed 08-05-2026. Pre-fix uploads had race silently lost; Jhotan + Japlan were restored manually.
- **Parser skill-checkbox bug**: `dnd/upload.js` marks all 18 skills proficient regardless of actual PDF checkbox state. Caught on Sage Greenwood (17-05-2026); polished manually. Worth diffing Grosh/Shieka/Jhotan/Japlan against their PDFs to see if they were over-marked at upload. Fix the parser.
- **Parser ability-score box reads modifiers**: players commonly fill the score boxes with the modifier (e.g. `+5` instead of `20`). Parser stores raw value; polish step should detect mod-vs-score and convert (heuristic: any value < 5 is almost certainly a mod; cross-check against weapon atk/dmg fields).
- **Vercel functions** in `api/upload.js` and `api/update.js` need an explicit `committer`/`author` payload to commit as `maltysnack` rather than the PAT owner's profile. Already done; if removed, upload PRs revert to showing the PAT owner's display name.
- **`dnd/.claude/launch.json`** is occasionally untracked and re-appears as a warning in sanitize. Safe to ignore.

## Things that would surprise the next session

- **dnd live sheet state is localStorage per-browser-per-device.** Switching devices = manual Export/Import (buttons removed but flow could be added back).
- **Sheet engine renders skills in a CSS grid.** Reading `innerText` on the page returns skills column-wise, which makes mods look misaligned with names; inspect the DOM rows directly (`li > .check-mod / .check-name / .check-ability`) when smoketesting.
- **The build action regenerates index.json + shims on merge.** Don't commit local-built shim or modified `dnd/characters/index.json` on a polish PR; revert before push so the action stays the source of truth on main.
- **PDF buttons depend on lazy-loaded pdf-lib.** First click has a ~2s pause while it fetches; subsequent clicks are instant.
- **Co-author trailer should be `Co-Authored-By: Wren <wren@maltysnack.dev>`**, not Claude.
