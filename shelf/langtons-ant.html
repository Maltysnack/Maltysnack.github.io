Langton's Ant

a two-dimensional Turing machine, 1986

Two rules. If the ant lands on a white cell, turn right, flip it black, step forward. If it lands on a black cell, turn left, flip it white, step forward. Repeat.

For the first ten thousand steps or so the pattern looks like noise, a tangle of flipped cells with no apparent structure. Then, without warning, the ant starts walking in a diagonal line and does not stop. It builds a repeating corridor, 104 steps per cycle, and extends it one tile at a time, indefinitely. The image below is the state after 11,500 steps. The highway is in the upper-right.

Nobody has proved why this happens. The two rules say nothing about a highway. The blank starting grid says nothing about a highway. It arrives out of apparent chaos and is the same highway every time.

201 x 201 grid, 11,500 steps from a single white cell

The whole simulation in JavaScript: 

const W = 201, H = 201;
let x = 100, y = 100, dir = 0; // 0=N 1=E 2=S 3=W
const dx = [0, 1, 0, -1], dy = [-1, 0, 1, 0];
const grid = new Uint8Array(W * H); // 0 = white

for (let i = 0; i < 11500; i++) {
  const idx = y * W + x;
  dir = grid[idx] === 0
    ? (dir + 1) & 3   // white: turn right
    : (dir + 3) & 3;  // black: turn left
  grid[idx] ^= 1;
  x = (x + dx[dir] + W) % W;
  y = (y + dy[dir] + H) % H;
}

Thirteen lines. The highway is not in those lines. It is in the interaction of those lines with the grid they make, which is the kind of thing that is hard to say more precisely than that.
