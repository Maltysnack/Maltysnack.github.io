Rule 30

an elementary cellular automaton, 1983

Imagine a row of cells, each black or white. To make the next row, look at every cell and its two neighbors. Three cells have eight possible patterns. A rule is just a table: for each pattern, is the cell below black or white? Eight bits, so 256 rules total.

Stephen Wolfram numbered them. 30 is the rule whose table, written as a binary byte, is 00011110. Start from a single black cell on an empty row, apply rule 30 again and again, and you get the pattern below. The left side is regular. The right side decays into stripes. The middle never repeats.

That center column is so apparently random that it was used as the random number generator inside Mathematica for years. From eight bits of rule and one black cell. No noise, no seed beyond position zero. Order's edge, generated by the simplest possible machine.

301 cells wide, 150 generations

The whole thing in JavaScript: 

// rule 30 = 00011110: pattern index 0..7 maps to bit (rule >> i) & 1
const w = 301, h = 150;
let row = new Uint8Array(w);
row[w >> 1] = 1;

for (let y = 0; y < h; y++) {
  draw(row, y);
  const next = new Uint8Array(w);
  for (let x = 0; x < w; x++) {
    const l = row[(x - 1 + w) % w];
    const c = row[x];
    const r = row[(x + 1) % w];
    next[x] = (30 >> ((l << 2) | (c << 1) | r)) & 1;
  }
  row = next;
}

Twelve lines. The pattern below is the output of those twelve lines, computed in your browser as the page loaded. Nothing was sent over the wire except the rule.
