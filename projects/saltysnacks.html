Project

Saltysnacks

Chrome extension. Replaces recipe pages with a clean two-column view: ingredients left, method right.

Manifest V3vanilla JSv2.004-05-2026

How it works

Reads Recipe JSON-LD if present, otherwise scrapes headings called Ingredients and Method. Imperial units convert to metric on the way in. A view original button restores the source page for that URL.

Features

  • Ingredient linksWords in the method that match an ingredient are tappable. Tap to see the amount; the matching line on the left highlights and scrolls into view.
  • Long-pressHold any word to add or remove it as a highlight. Choices sync via chrome.storage.sync.
  • Send to phoneGenerates a self-contained HTML file and hands it to the native share sheet.
  • PrintCollapses to a single column for paper.
  • Overlay cleanupMutationObserver removes fixed and sticky elements the original page reinjects after takeover.

The numbered-section bug

The DOM-fallback parser treated any heading whose text contained the substring step as a recipe method. So a tutorial titled Step 1: install the package matched, and the next <ol> got rendered as a recipe.

Fix: require an exact heading word (Method, Instructions, Directions, Steps), and require a sibling ingredient list before the fallback fires.
