01 Lexical structure

Source is plain ASCII. Tokens are separated by whitespace (space, tab, newline). Whitespace is otherwise insignificant.

Identifiers

IDENTIFIER → [a-zA-Z][a-zA-Z0-9_]*

Used as river, dam, terminator, and system names. Case-sensitive.

Numbers

NUMBER → [0-9]+ ( "." [0-9]+ )?

Decimal literals, optional fractional part.

02 Grammar

program → statement* EOF statement → systemDecl | listStmt | mapStmt | printStmt | chainDecl systemDecl → "system" IDENTIFIER ";" chainDecl → node ( "->" node )* ";" node → river | dam | terminator river → IDENTIFIER "[" NUMBER "," NUMBER ( "," NUMBER )? "]" dam → IDENTIFIER "{" NUMBER "," percent "," NUMBER "," NUMBER "}" terminator → IDENTIFIER "[" "]" listStmt → "list" IDENTIFIER ";" mapStmt → "map" IDENTIFIER ";" printStmt → "print" IDENTIFIER NUMBER? ";" percent → NUMBER "%" | NUMBER

03 Statements

FormMeaning
system Name; Declares a named system. Subsequent chain declarations attach to the most recently declared system.
list Name; Prints every river, dam, and terminator in the named system.
map Name; Prints an ASCII diagram of the network for the named system.
print Name [days]; Simulates the system for the given number of days (default 1) and prints the per-day flow table and cumulative outlet volumes.
chain; A connected sequence of nodes joined by ->. Adds the nodes and edges to the current system.

04 Node types

River

Name[base, peak, decay?]
baseBaseline flow in litres per second with no rainfall.
peakMultiplier applied to rainfall (mm) to compute the peak surge.
decayFraction of the surge retained each subsequent day. Range 0 to 1.

Dam

Name{capacity, threshold, normalRate, openRate}
capacityMaximum storage in megalitres.
thresholdStorage level at which the dam opens. Written as a percent (80%) or fraction (0.8).
normalRateOutflow as a fraction of inflow when storage is below the threshold.
openRateOutflow as a fraction of inflow when storage is at or above the threshold.

Terminator

Name[]

An outlet. Accumulates everything that flows into it.

05 Operators

Flow arrow

A -> B

Directed flow from A to B. A node may appear in multiple chains; its parameters must match across declarations. If a node has multiple outgoing edges, flow is split evenly between them.

06 Semantics

River hydrograph

Given r millimetres of rainfall on day 0, river flow on day n is:

day 1 (rising): base + 0.3 × peak × r
day 2 (peak): base + peak × r
day n ≥ 3: base + peak × r × decay^(n - 2)

With no rainfall, every day is just base.

Dam outflow

If current storage is below threshold, outflow is inflow × normalRate. Otherwise outflow is inflow × openRate. Storage is updated each day by inflow − outflow.

Terminator accumulation

A terminator sums every unit of flow that arrives at it across the simulation. The total appears in the print output as the cumulative outlet volume.
