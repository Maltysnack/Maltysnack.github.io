01 Lexical structure

Source is plain ASCII. Whitespace separates tokens but is otherwise insignificant. Two comment forms are supported.

Identifiers

IDENTIFIER → [a-zA-Z][a-zA-Z0-9_]*

Used as system, river, dam, and outlet names. Case-sensitive.

Numbers

NUMBER → [0-9]+ ( "." [0-9]+ )?

Decimal literals, optional fractional part.

Comments

// begins a line comment that runs to the next newline. /* ... */ begins a block comment that may span multiple lines. Block comments do not nest.

";"? block → "{" ( attr ( ","? attr )* )? "}" attr → IDENTIFIER ":" NUMBER "%"? chain → IDENTIFIER ( "->" IDENTIFIER )* ";" listStmt → "list" IDENTIFIER ";" mapStmt → "map" IDENTIFIER ";" printStmt → "print" IDENTIFIER ( NUMBER | series )? 03 Statements

FormMeaning
system Name; Declares a named system. Subsequent declarations and chains attach to the most recently declared system.
river Name { ... } Declares a river node and its parameters.
dam Name { ... } Declares a dam node and its parameters.
outlet Name; Declares an outlet (zero-flow accumulator). Block is optional and must be empty.
A -> B -> C; Connects nodes by name. Names must be declared somewhere in the same system. Source order does not matter.
list Name; Lists every river, dam, and outlet in the named system.
map Name; Prints an ASCII diagram of the network.
print Name; Simulates the system over ten days at 1.0 mm rainfall on day 1, dry thereafter. Prints the per-day flow table and cumulative outlet volumes.
print Name r; Simulates with a single r mm rainfall event on day 1.
print Name [r1, r2, ...]; Simulates with a per-day rainfall series in millimetres. Each non-zero entry kicks off its own hydrograph; flow on day n is the superposition of every event so far. Series shorter than ten days is padded with zeros.
help; Prints a short syntax reference covering declarations, chains, and the print forms. Useful at the CLI; the playground duplicates most of it in the Blank template.

The interpreter runs in two passes. Pass 1 collects declarations and chains. A resolve step validates each chain reference against the per-system symbol table. Pass 2 executes list, map, and print in source order.

04 Node attributes

River

AttributeRequiredMeaning
baseyesBaseline flow in litres per second with no rainfall.
peakyesMultiplier applied to rainfall (mm) to compute the peak surge.
decaynoFraction of the surge retained each subsequent day. Range 0 to 1. Default 0.7.

Dam

AttributeRequiredMeaning
capacityyesMaximum storage in megalitres.
thresholdyesStorage level at which the dam opens. Written as 80% or 0.8.
normalRateyesOutflow as a fraction of inflow when below threshold.
openRateyesOutflow as a fraction of inflow when at or above threshold.

Outlet

No attributes. An outlet accumulates everything that flows into it. An outlet may have outgoing connections, in which case it acts as a zero-flow passthrough.

05 Block syntax

Attributes are written as name: value. Attributes may be separated by commas, whitespace, or both. Single-line and multi-line forms are both legal.

river Spring { base: 2, peak: 2.0, decay: 0.7 } river Spring { base: 2 peak: 2.0 decay: 0.7 }

The semicolon after a block is optional. After a bare outlet Name without a block, the semicolon is required.

Unknown, missing, or duplicate attributes are parse errors that point at the offending declaration.

06 Operators

Flow arrow

A -> B

Directed flow from A to B. A node may appear in any number of chains. If a node has multiple outgoing edges, flow is split evenly between them.

07 Semantics

River hydrograph

Given r millimetres of rainfall on day 0, river flow on day n is:

day 1 (rising): base + 0.3 × peak × r
day 2 (peak): base + peak × r
day n ≥ 3: base + peak × r × decay^(n - 2)

With no rainfall, every day is just base.

Dam outflow

If current storage is below threshold, outflow is inflow × normalRate. Otherwise outflow is inflow × openRate. Storage is updated each day by inflow − outflow.

Outlet accumulation

An outlet sums every unit of flow that arrives at it across the simulation. The total appears in the print output as the cumulative outlet volume.
