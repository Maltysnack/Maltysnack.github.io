Flox

A small language for describing flow networks. You write nodes and arrows; it simulates how stuff moves through them over time. The same parser and engine handle several kinds of network; mode picks which.

Flox is a stylized model, not a hydrology engine, not a power-flow solver, not a probability-theory textbook. Good for sketching the shape of a system, not for forecasting.

Modes

Modes share the same syntax (system, declarations, chains, print) but apply different physics. Each card below is a snippet; the full version runs in the playground.

River

live

Water flows downstream through rivers, dams, and outlets, driven by per-day rainfall. The default mode; programs without a mode line are river programs. 

system Demo;

Spring    -> Confluence -> Outlet;
Tributary -> Confluence;

river  Spring     { base: 2, peak: 2.0, decay: 0.7 }
river  Tributary  { base: 1, peak: 1.5, decay: 0.6 }
river  Confluence { base: 3, peak: 1.8 }
outlet Outlet;

print Demo 5;

Markov

live

States redistribute probability per-tick across weighted edges. Cycles work; outgoing weights from each state must sum to 1. Useful for any "what's the long-run distribution of this process?" question. The example: a footballer's pass-or-shoot decision in the attacking third, calibrated on ~10% league-average shot conversion. 

mode markov;
system PassOrShoot;

Pass  -[0.85]-> Pass;
Pass  -[0.15]-> Shoot;
Shoot -[0.10]-> Goal;
Shoot -[0.90]-> Miss;
Goal  -[1.0]-> Goal;
Miss  -[1.0]-> Miss;

node Pass;  node Shoot;  node Goal;  node Miss;

start Pass 1.0;
print PassOrShoot 200;

Grid

live

An interconnected web. Nodes inject (generators) or draw (loads); edges conduct between them, bidirectionally. Levels equilibrate to satisfy Kirchhoff's law at every node. Useful for power grids and traffic networks. Total supply must sum to zero (generators balance loads).

disable takes a node or edge out for one run. compare runs the same system under several scenarios side by side. The example below answers "what happens if the main plant trips, or the tie line is cut?" in one pass. 

mode grid;
system PowerGrid;

Plant     -[5]-> Hub;
Backup    -[2]-> Hub;
Hub       -[3]-> NorthCity;
Hub       -[3]-> SouthCity;
NorthCity -[1]-> SouthCity;

node Plant     { supply:  100 }
node Backup    { supply:   30 }
node Hub;
node NorthCity { supply:  -65 }
node SouthCity { supply:  -65 }

compare PowerGrid 500 as
    "Baseline",
    "Plant offline" with disable Plant,
    "Tie line cut"  with disable NorthCity -> SouthCity;

Coming

planned

Each mode below is a directed-graph simulator with its own per-tick rule. The engine is mode-agnostic, so adding one is mostly defining its tick function and printer.

  • Plumbing: pressure-driven flow with friction.
  • Traffic: cars through intersections with arrival rates and capacities.
  • SIR: epidemic compartmental model (susceptible / infectious / recovered).
  • Supply: factories, warehouses, stores; lead times and stockouts.
