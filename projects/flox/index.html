Project

Flox

A small language for describing flow networks. You write nodes and arrows; it simulates how stuff moves through them over time.

Flox is a stylized model, not a hydrology engine. Good for sketching the shape of a system, not for forecasting.

v0.2Java08-05-2026

Example

Program Try it yourself in the playground 
// A small two-source demo system.
// Two upland creeks meet at a confluence and flow to an outlet.
system Demo;

// Chains: pure references, so the network shape reads at a glance.
Spring    -> Confluence -> Outlet;
Tributary -> Confluence;

/* Declarations: rich blocks, order doesn't matter.
   River params:
     base  - baseline flow (L/s) with no rainfall
     peak  - multiplier on rainfall (mm) for the peak surge
     decay - fraction of the surge retained each subsequent day */

river  Spring     { base: 2, peak: 2.0, decay: 0.7 }
river  Tributary  { base: 1, peak: 1.5, decay: 0.6 }
river  Confluence { base: 3, peak: 1.8 }
outlet Outlet;

map Demo;
print Demo 5;   // simulate at 5mm of rainfall

Chains list the network shape with bare names. Declarations follow as river, dam, or outlet blocks. -> is directed flow. print Demo 5 runs the simulation at 5mm of rainfall on day 1.

Rainfall as a series

A scalar after print is one rainfall event on day 1. To model a real weather pattern, pass a per-day series in millimetres: 

river  Spring { base: 2, peak: 2.0, decay: 0.7 }
outlet Outlet;
Spring -> Outlet;

print SeriesDemo [0, 1, 30, 10, 0, 0, 0];   // dry, light rain, storm, light, dry

Each non-zero day kicks off its own rising-peak-decay hydrograph. Flow on day n is the superposition of every event so far. Below, day 3's 22.0 is the peak from day 2's 1mm event plus the rising limb of day 3's 30mm storm. Day 4's 69.4 is the storm peak.
