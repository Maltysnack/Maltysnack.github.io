Project

Flox

A small language for describing flow networks. You write nodes and arrows; it simulates how stuff moves through them over time.

v0.1Javaprivate repo06-05-2026

Example

system Canberra;

Googong[2, 2.0, 0.7] -> googongDam{1000, 80%, 0.8, 1.2} -> Queanbeyan[3, 1.5];
upperMongolo[3, 1.8, 0.6] -> Queanbeyan[3, 1.5];
Queanbeyan[3, 1.5] -> centralMongolo[];
centralMongolo[] -> burleyGriffinDam{5000, 60%, 0.9, 1.5} -> lowerMongolo[];

map Canberra;
print Canberra 10;

Square brackets are rivers. Curly brackets are dams. Empty brackets are outlets. -> means flow from left to right. print Canberra 10 simulates ten days and prints the flow table.

Status

WorksScanner, parser, AST, tree-walking interpreter. River hydrograph simulation. Network map. Multi-day flow tables.
BrokenDam units bug: inflow (L/s) and storage (ML) are mixed, so threshold is never reached and dam dynamics are effectively stubbed.
MissingComments. Named parameters. Declare-once references. Rainfall as a series. Tests.

Roadmap

  1. Harden the river language. Comments, named parameters, declare-once references, fix the dam units bug, rainfall as a series, golden-file tests.
  2. Port the interpreter to TypeScript. Browser playground with Monaco editor, D3 graph, Recharts hydrograph, shareable permalinks.
  3. Generalise the engine. mode switch, edge weights, cycles. First non-river demo: a Markov chain.
  4. Three.js renderer. Procedural terrain, splined rivers, dam geometries.
  5. More modes. Traffic, SIR epidemics, supply chains, pressure-driven plumbing.
