Bauhaus

Generative Bauhaus compositions, rendered live in SVG. No model training, just the school's own visual grammar: primary palette, geometric primitives, asymmetric balance on a grid.

Path

Plan, in order. Each step is small and ships before the next begins.

  1. 01Page wiring. This page exists, sits in the sidebar, and renders correctly on GitHub Pages. Done.
  2. 02First generator. A JS module that places a circle, square, triangle, and a few thin bars on a 12-column grid using a seedable random. Renders a fresh composition every load.
  3. 03Composition rules. Add asymmetric balance (visual centre of mass within bounds), palette weighting (yellow rare, blue calm, red loud), and a grid that primitives must snap to.
  4. 04Variations. A small set of named rule-sets, each one referencing a different studio mode: Itten colour study, Klee tile, Albers nested squares, Kandinsky overlap.
  5. 05Permalink by seed. Every composition is reproducible from a short seed in the URL. No backend needed.

Notes

No ML. The Bauhaus visual grammar is small and explicit. Rules give a faster, cleaner result than a trained model on this scale.

No build step. Plain HTML and ES modules, the same as the rest of the site. No Vite, no bundler.

No backend. Pure client-side SVG. GitHub Pages serves the whole thing. Supabase and Vercel are not needed unless we later want a saved gallery of seeds.
